# 每日规划

## 核心框架

每天只需要确定两条主线：
- 🏢 **工作主线**（默认6点前）：今天工作上做成什么算没白过
- 🌙 **个人主线**（默认6点后）：今天个人事项做成什么算没白过

主线强制**每桶只有一条**。

固定日常活动（健身、跑步、吃药等）可以列在对应时段下，不占主线名额。

其余任务清单是背景噪音，有空再搭，不强求。

## 执行流程

### Step 0：昨日回顾（如果昨天笔记里有主线）

1. 打开昨天的笔记，看两条主线是否完成。
2. 一次性问用户：
   > 主线完成了吗？没完成的话一句话说下原因就行。
3. 把回顾写到**昨天笔记的末尾**（封档），格式：

   ```
   ---
   **昨日回顾**
   * 🏢 [工作主线]：✓ / ✗ 未完成
   * 🌙 [个人主线]：✓ / ✗ 未完成
   * 原因：[一句话]（仅未完成时写）
   ```

4. 只回顾主线，不回顾其他 backlog 任务。

### Step 1：迁移与重构

1. 读取昨天的笔记，把未完成的任务迁到今天的笔记。
2. 重构 todo 结构：**每一层最多 3 个子节点，超了就再分一层**（递归规则）。
   - 按性质分组：顶层节点按性质分类（例如"数据 / 训练"、"修 / 补"、"读 / 写"）。
   - 超过 3 个说明还没抽象到位，需要再抽一层分类把它们归拢。
   - 归并原则：本质相同的合成一类；不是本轮重点的降级或删除。
   - 改完后展示给用户确认，再写回文件。

### Step 2：定主线

1. 一次性问：
   > 如果今天结束，你只做成了一件工作上的事、一件自己的事，各是什么会让你觉得"今天没白过"？

2. 收到回答后，直接将主线写入今天日记顶部：

   ```
   **今日主线**
   * 🏢 工作（before 6pm）：[工作主线]
   * 🌙 个人（after 6pm）：[个人主线]
   ```

   如果用户还提到了固定日常活动，加在对应时段下作为子项：

   ```
   * 🌙 个人（after 6pm）：
       * 健身
       * [个人主线]
   ```

## 原则

- 每步只问一个问题，保持简短
- 主线一桶一条，不迁就
- 昨日回顾只看主线，其他任务不做形式化总结
- 不建议优先级、不拆解子任务（拆解交给用户）
- 写入日记后即完成，不做额外总结
