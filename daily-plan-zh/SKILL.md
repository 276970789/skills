# 每日规划

## 核心框架

每天只需要确定两条主线：
- 🏢 **工作主线**（默认6点前）：今天工作上做成什么算没白过
- 🌙 **个人主线**（默认6点后）：今天个人事项做成什么算没白过

其余任务清单是背景噪音，有空再搭，不强求。

## 执行流程

1. 读取当天日记，了解现有任务列表。

2. 重构 todo 结构：**按性质分组**。
   - 顶层节点按性质分类（例如"数据 / 训练"、"输入 / 输出"、"读 / 写"）。
   - 顶层分类硬约束：**同一父节点下的顶层分类最多 3 个**；超过说明性质太杂，需要归并。
   - 同类下的细分项不限数量（本来就是一件事的细节）。
   - 归并原则：几条本质相同的合成一类；不是本轮重点的降级或删除。
   - 改完后展示给用户确认，再写回文件。

3. 一次性问：
   > 如果今天结束，你只做成了一件工作上的事、一件自己的事，各是什么会让你觉得"今天没白过"？

4. 收到回答后，不再追问，直接将主线写入日记顶部：

   ```
   **今日主线**
   * 🏢 工作（before 6pm）：[工作主线]
   * 🌙 个人（after 6pm）：[个人主线]
   ```

## 原则

- 只问一个问题，保持简短
- 不建议优先级、不拆解子任务（拆解交给用户）
- 写入日记后即完成，不做额外总结
